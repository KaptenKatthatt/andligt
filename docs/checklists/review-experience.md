# Review Experience Checklist

## Purpose

Every change to Visdomsatlasen should make the experience calmer, clearer and more meaningful.

This checklist is not about code quality.

It is about protecting the experience.

If a feature makes the application more distracting, more demanding or more complicated, it should be reconsidered.

---

# 1. First Impression

Open the app as if you have never seen it before.

Ask yourself:

- [ ] Does the app immediately feel calm?
- [ ] Does the interface invite me to slow down?
- [ ] Is there a clear starting point?
- [ ] Does the screen feel spacious rather than crowded?

---

# 2. Cognitive Load

Every decision requires mental energy.

Ask yourself:

- [ ] Have we reduced unnecessary choices?
- [ ] Does every visible element serve a purpose?
- [ ] Could something be removed?
- [ ] Is there anything competing for attention?

---

# 3. Reading Experience

The reading experience is the heart of Visdomsatlasen.

- [ ] The text is always the primary focus.
- [ ] Typography is comfortable.
- [ ] Line length is easy to read.
- [ ] Spacing supports slow reading.
- [ ] Nothing interrupts the reader unnecessarily.

---

# 4. Interaction

Interactions should feel invisible.

- [ ] Navigation is obvious.
- [ ] Buttons are few and intentional.
- [ ] There are no unnecessary interactions.
- [ ] Every interaction has a clear purpose.

---

# 5. Pace

Visdomsatlasen should never feel rushed.

- [ ] The interface encourages slow reading.
- [ ] Nothing pushes the user forward.
- [ ] The user never feels pressured to continue.
- [ ] The room feels complete on its own.

---

# 6. Visual Noise

Remove anything that competes with the content.

- [ ] No decorative elements without purpose.
- [ ] No unnecessary icons.
- [ ] No distracting animations.
- [ ] No visual clutter.
- [ ] Plenty of whitespace.

---

# 7. Emotional Experience

Ask yourself how the app feels.

- [ ] Peaceful
- [ ] Respectful
- [ ] Thoughtful
- [ ] Timeless
- [ ] Quiet

If any answer is **No**, identify what creates the wrong feeling.

---

# 8. The "Library Test"

Imagine this screen inside a quiet library.

Ask yourself:

- [ ] Would this interface feel at home there?
- [ ] Would anything feel too loud?
- [ ] Would anything break the atmosphere?

If yes, simplify it.

---

# 9. The "One Room Test"

Imagine the user only visits one room today.

Ask yourself:

- [ ] Would that still feel like a complete experience?
- [ ] Does the app avoid encouraging more consumption?
- [ ] Does the room stand on its own?

---

# 10. The "Remove Something" Test

Before approving any change, remove one element.

Ask yourself:

- [ ] Is the experience now worse?

If the answer is **No**, that element probably wasn't needed.

Keep simplifying until every remaining element has earned its place.

---

# Final Reflection

Before merging a feature, answer these questions honestly.

- [ ] Does this make Visdomsatlasen calmer?
- [ ] Does this help the user think more deeply?
- [ ] Does this reduce friction without increasing distraction?
- [ ] Does this respect the user's attention?
- [ ] Would I still build this feature if engagement metrics did not exist?

If any answer is **No**, rethink the design before proceeding.
